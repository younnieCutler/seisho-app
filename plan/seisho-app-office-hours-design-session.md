# 세이쇼 앱 Office Hours 설계 세션

## 세션 개요

- **날짜**: 2026-03-24
- **모드**: Builder (사이드 프로젝트)
- **목표**: 40개 기능 명세 → MVP 범위 확정 + 설계 문서 작성
- **결과**: A+C 접근방식 확정, 8개 핵심 기능으로 스코프 축소

---

## 전제 검토 결과

| 전제                     | 결론                                          |
| ---------------------- | ------------------------------------------- |
| 절기 맥락 수요               | Domain insider 인사이트 (20년+ 교회 경험). 미검증이지만 타당 |
| AI 해석 = 핵심 차별점         | ❌ Nice-to-have로 수정. v2 이연                   |
| 40개 기능 × 1~2개월         | ❌ 불가능 → 8개로 축소                              |
| Gamification = 핵심 동기부여 | ✅ 인정                                        |

---

## 확정된 방향: A + C 조합

### 핵심 인사이트
기존 한국 성경 앱(매일성경, 생명의삶)은 개인 읽기 중심.
**교회 절기 맥락 구조화 + 소그룹 공동 루틴**이 진짜 갭.

### v1 핵심 8개 기능

| # | 기능 | 복잡도 |
|---|------|--------|
| 1 | 절기 자동 감지 알고리즘 | M |
| 2 | 오늘의 말씀 화면 | S |
| 3 | 읽기 완료 체크 | S |
| 4 | 잔디 그래프 (개인) | M |
| 5 | 소그룹 코드 생성/참여 | M |
| 6 | 소그룹 피드 (1문장 나눔) | M |
| 7 | 연속 스트릭 표시 | S |
| 8 | 온보딩 (연령대만, 한국어 전용) | S |

### v2로 이연
AI 해석, 홈 화면 위젯, 인앱결제, 관리자 대시보드, 다국어, 묵상 노트, 푸시 알림

---

## 기술 스택

- **프론트엔드**: React Native (Expo) — iOS/Android 동시 개발
- **백엔드**: Supabase (익명 인증, 실시간 DB, 무료 티어 v1)
- **절기 데이터**: 로컬 JSON (교회력 2025-2027, 8개 절기)
- **성경 데이터**: 큐레이션 JSON (하단 참조)

---

## 성경 데이터 전략 (중요 결정)

### 문제
개역개정 성경 = 대한성서공회 저작권 → api.bible 무료 플랜 포함 불가.

### 핵심 인사이트
이 앱은 "전체 성경 읽기 앱"이 아님. **"절기별 오늘의 말씀"** 앱.
전체 성경 API 불필요 — 절기별 큐레이션 구절 약 200개만 있으면 충분.

### 결정된 전략
1. **v1**: 절기별 말씀 ~200구절 → 앱 내장 JSON (직접 큐레이션)
2. **체류 시간 증가를 위한 본문 읽기**: "이 말씀이 속한 장 읽기" → YouVersion 외부 링크 또는 YouVersion API (무료, 개역개정 포함 가능성 확인 필요)
3. **수익화 전**: 대한성서공회 라이선스 취득 필수

### api.bible 의존성 완전 제거
- 비용 0, 외부 장애 없음, 오프라인 기본 지원

---

## 아키텍처 결정사항

### 인증
- v1: Supabase 익명 인증 + AsyncStorage device ID 지속
- 재설치 시 데이터 유실 허용 (MVP 단순화)
- v2: 이메일 로그인 + 기기 간 동기화

### 오프라인
- 말씀: 앱 내장 JSON (항상 오프라인 가능)
- 완료 체크 + 잔디: 로컬 저장 → 온라인 시 Supabase 동기화
- 충돌: last-write-wins (UTC timestamp)

### 소그룹 코드
- 6자리 영숫자 랜덤 (예: `AB12XY`)
- 잘못 입력 5회 → 10분 잠금
- 재사용 불가 (soft-delete)

---

## 성공 기준 (v1, 배포 4주 후)

- WAU ≥ 10명 (한 주에 오늘의 말씀 화면 ≥1회 조회한 고유 사용자)
- 루틴 완료율 ≥ 40% (주 3일 이상 체크)
- 소그룹 나눔 참여율 ≥ 30%

---

## 빌드 순서

1. 절기 데이터 JSON 생성 (2025~2027 교회력 + 절기별 말씀 큐레이션)
2. Expo 프로젝트 + Supabase 초기화
3. 홈 화면 (절기 표시 + 오늘의 말씀)
4. 완료 체크 + 잔디 그래프
5. 소그룹 코드 + 피드
6. 온보딩
