# Seisho-App v1 설계 문서

> Generated by /office-hours · 2026-03-24 · Status: APPROVED

---

## 문제 정의

대한예수교장로회 20~30대 청년 신자들이 평일에 꾸준한 성경 묵상 루틴을 유지하기 어렵다.
특히 "주일에만 독실한 기독교인"이 되는 경향이 있고, 절기의 의미를 일상에서 연결하는 도구가 없다.

기존 앱(매일성경, 생명의삶)은 개인 읽기 중심이며, 교회 절기 맥락을 구조화하거나
소그룹과 함께하는 공동 루틴을 지원하지 않는다.

---

## 핵심 차별점

1. **절기 맥락 자동화** — 앱이 오늘이 사순절 며칠째인지, 대림절인지 알고 콘텐츠를 바꾼다.
2. **소그룹 공동 루틴** — 교회 소그룹 코드 하나로 같은 말씀을 보고, 1문장씩 나눈다.
3. **잔디 그래프** — 개인 기록이 시각적으로 쌓인다. 신앙의 흔적이 눈에 보인다.

---

## 선택된 방향: A + C

**절기 루틴 코어(A) + 소그룹 나눔(C) 조합**

|---|------|--------|

## 기술 스택

| 레이어 | 선택 | 이유 |
|--------|------|------|
| 프론트엔드 | React Native (Expo) | iOS/Android 동시 지원 |
| 백엔드 | Supabase | 무료 티어, realtime (소그룹 피드) |
| 인증 | Supabase 익명 인증 | 로그인 없이 바로 사용 |
| 성경 데이터 | SQLite 번들 (`ko.db`, expo-sqlite) | JSON 파싱 없음, 구절 쿼리 빠름 |
| 유저 데이터 | react-native-mmkv | AsyncStorage 대비 성능 우수 |
| 절기 데이터 | 로컬 JSON | 2025~2030 교회력 미리 생성 |

## 성경 데이터 전략 (최종)

**외부 API 의존성 없음. 앱 내 로컬 번들.**

| 번역본 | 라이선스 | v1 사용 |
|--------|---------|---------|
| Open Bible ko-KR | CC BY-SA 4.0 | ✓ **1순위** |
| 개역한글 (Kaggle) | Public Domain 확인됨 (저작권 만료) | ✓ **2순위** |
| 개역개정 | 대한성서공회 유료 | ✗ v1 불가 (v2 라이선스 취득 후) |

**v2+ 다국어:**
- 일본어: `christos-c/bible-corpus jpn.xml` (Public Domain 1955판)
- 영어: WEB (World English Bible, Public Domain)

```
/assets/bible/
  ko.db   ← Open Bible ko-KR SQLite 번들 (v1)
  ja.db   ← v2
  en.db   ← v2
/assets/liturgy/
  calendar-2025-2030.json
```

**CC BY-SA 4.0 표기**: 앱 내 설정 > 오픈소스 라이선스 섹션에 출처 명시.

## 소그룹 설계

- **코드**: 6자리 영숫자 (`AB12XY`), 전역 유니크, 재사용 불가
- **리더**: 코드 생성자 = 리더 (멤버 보기 권한, 추방 v2)
- **나눔**: 비공개 (코드 보유자만 피드 열람), 150자 제한
- **보안**: 5회 오입력 → 10분 잠금

---

## 인증 & 오프라인

- **인증**: Supabase 익명 인증 → MMKV에 ID 저장 → 앱 재시작 시 유지
- **오프라인**: 성경 본문은 SQLite 번들 (앱 내 영구 포함), 유저 진도/스트릭은 MMKV 저장 후 Supabase 동기화
- **동기화**: last-write-wins (UTC timestamp) — 1인 1기기 가정, 충분함

---

## 성공 기준 (v1 배포 후 4주)

- WAU 10명 이상
- 루틴 완료율 40% 이상 (주 3일 이상 체크)
- 소그룹 나눔 참여율 30% 이상

---

## 빌드 순서

1. **성경 데이터 준비** — Open Bible ko-KR 다운로드 → SQLite `.db` 변환 (expo-sqlite), 절기별 구절 ~200개 큐레이션 (RCL 기준), 2025~2030 교회력 JSON
2. **Expo 초기화** — React Native + Supabase + i18n 구조
3. **홈 화면** — 오늘의 절기 + 말씀
4. **완료 체크 + 잔디 그래프**
5. **소그룹 코드 + 피드**
6. **온보딩 플로우**
