## MVP v1 기능 범위

1. **성경 본문** — api.bible 연동, 책/장/절 검색, 즐겨찾기, 공유, 폰트 조절
2. **AI 묵상 코치** — Gemini Flash 묵상 생성 (절기 인식), Claude Haiku 구절 Q&A
3. **절기 캘린더** — 사순절/부활절/대림절 자동 인식, 캘린더 UI
4. **일독 진도** — 목표 설정, 진도 기록, 시각화, 다기기 동기화
5. **사용자 계정** — Google/Apple 소셜 로그인 + 이메일, Supabase Auth
6. **푸시 알림** — 매일 아침 절기 맞춤 구절 + AI 요약
7. **묵상 저널링** — AI 묵상 후 개인 메모 저장, 기록 조회
8. **오프라인 캐싱** — 성경 전체 Expo SQLite 로컬 저장 (첫 실행 시 다운로드)

**v1.1 이동:** 홈 화면 위젯

**NOT in scope:** 가톨릭 절기, 영어 전용 글로벌, B2B 라이선스, 설교 요약 AI

## 데이터베이스 스키마 (Supabase PostgreSQL)

```sql
-- 사용자 프로필 (auth.users INSERT 트리거로 자동 생성)
CREATE TABLE users (
  id uuid PRIMARY KEY,        -- Supabase Auth UUID
  display_name text,
  pref_lang_ui text DEFAULT 'ko',     -- UI 언어
  pref_lang_bible text DEFAULT 'ko',  -- 성경 본문 언어
  pref_lang_ai text DEFAULT 'ko',     -- AI 묵상 언어
  created_at timestamptz DEFAULT now()
);

-- 즐겨찾기 구절
CREATE TABLE bookmarks (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES users(id) ON DELETE CASCADE,
  bible_id text NOT NULL,   -- api.bible 버전 ID
  book_id text NOT NULL,
  chapter int NOT NULL,
  verse_start int NOT NULL,
  verse_end int NOT NULL,
  created_at timestamptz DEFAULT now()
);

-- 일독 플랜
CREATE TABLE reading_plans (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES users(id) ON DELETE CASCADE,
  plan_type text NOT NULL,  -- '1year' | '6month' | 'custom'
  start_date date NOT NULL,
  target_end_date date NOT NULL,
  created_at timestamptz DEFAULT now()
);

-- 일독 진도 (Last Write Wins — upsert on UNIQUE)
CREATE TABLE reading_progress (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES users(id) ON DELETE CASCADE,
  plan_id uuid REFERENCES reading_plans(id) ON DELETE CASCADE,
  book_id text NOT NULL,
  chapter int NOT NULL,
  read_at timestamptz DEFAULT now(),
  UNIQUE (plan_id, book_id, chapter)
);
CREATE INDEX idx_reading_progress_user_plan ON reading_progress(user_id, plan_id);

-- AI 묵상 캐시 (구절+절기 복합 UNIQUE — 중복 생성 방지)
CREATE TABLE meditation_cache (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  bible_ref text NOT NULL,          -- 'GEN.1.1-GEN.1.3'
  liturgical_season text NOT NULL,  -- 'lent' | 'easter' | 'advent' | 'christmas' | 'ordinary'
  content_ko text,                  -- Lazy fill: 요청 언어만 먼저 저장
  content_ja text,                  -- null = 아직 미생성
  content_en text,                  -- null = 아직 미생성
  generated_at timestamptz DEFAULT now(),
  UNIQUE (bible_ref, liturgical_season)
);

-- 묵상 저널 (개인 메모)
CREATE TABLE reflections (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES users(id) ON DELETE CASCADE,
  bible_ref text NOT NULL,
  content text NOT NULL,
  meditation_id uuid REFERENCES meditation_cache(id),
  created_at timestamptz DEFAULT now()
);

-- 후원 기록
CREATE TABLE donations (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES users(id),
  amount_krw int NOT NULL,
  message text,
  created_at timestamptz DEFAULT now()
);
```

### RLS 정책 (모든 테이블 적용)
```sql
-- 예: bookmarks
ALTER TABLE bookmarks ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can only access own bookmarks"
  ON bookmarks FOR ALL
  USING (auth.uid() = user_id);
```

### auth.users 자동 동기화 트리거
```sql
CREATE OR REPLACE FUNCTION handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.users (id, display_name, pref_lang_ui, pref_lang_bible, pref_lang_ai)
  VALUES (NEW.id, NEW.raw_user_meta_data->>'full_name', 'ko', 'ko', 'ko');
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE PROCEDURE handle_new_user();
```

## AI 전략 (비용 최적화)

### 묵상 생성 — Lazy Fill
```
요청(bible_ref, season, lang)
  → meditation_cache 확인
  → 히트 + lang 컬럼 존재: 캐시 반환 (API 비용 0)
  → 히트 + lang 컬럼 null: Gemini Flash → lang 컬럼만 UPDATE
  → 미스: Gemini Flash → lang 컬럼만 INSERT (나머지 null)
```

### Q&A — 서버사이드 한도 강제
```typescript
// Edge Function /ai-qa
const usage = await getUsageCount(userId, today)
if (usage >= 5 && !isDonor(userId)) {
  return { error: 'DAILY_LIMIT_REACHED', limit: 5 }
}
// Claude Haiku 호출 (API 키는 서버에만 존재)
```

```sql
-- 매일 오전 7시 KST (22:00 UTC)
SELECT cron.schedule('daily-push', '0 22 * * *',
  $$SELECT net.http_post(url := 'https://<project>.supabase.co/functions/v1/daily-push',
    headers := '{"Authorization": "Bearer <service-key>"}'::jsonb)$$
);
```

# 📌 [추가 문서] 기획 고도화 리뷰 (Beachhead & Product Discovery)

기존 구현 계획서 (위 내용)를 바탕으로 초기 시장 침투 전략 및 기능을 확장한 추가 기획 제안입니다.

## 1. Beachhead Segment 분석 (초기 거점 시장)

`@[/beachhead-segment]` 워크플로우를 활용하여 제품 출시 시 가장 적합한 초기 거점 시장을 분석했습니다.

### 1-1. 최우선 추천 거점 세그먼트
**한국 개신교 중소 교회 (10명 ~ 30명 규모의 소그룹/교회)**
- **선정 이유**: 높은 Burning Pain (자체 디지털 자료 부족), 확고한 그룹 네트워크를 통한 바이럴(Referral) 잠재력, 교회/교육 예산을 활용한 B2B2C 구독형 모델 가능성.
- **Winnable Share**: 대형 교회는 자체 앱이 있지만, 중소 교회는 대체재가 부족해 60~70% 선점을 빠르게 달성할 수 있습니다.

### 1-2. 보조 후보 세그먼트
**일본 거주 한인 교인 네트워크**
- 초기부터 i18n이 설계되므로, 향후 일본 내 한인 교회에서 빠르게 바이럴될 수 있는 잠재력이 높습니다.

## 2. Product Trio 브레인스토밍 (Product Discovery)

핵심 타겟의 **리텐션, 바이럴, 인게이지먼트 극대화**를 목표로 도출된 3가지 관점의 기능 아이디어입니다.

### 2-1. 3가지 관점의 아이디어 15선

**🎯 Product Manager (비즈니스 가치, 리텐션, 성장에 초점)**
1. **소그룹 공동 일독 플랜 공유**: 초대 링크를 통한 교회/소그룹 단위의 읽기 진도 공유 (Referral)
2. **연속 묵상(Streak) 시스템**: 매일 묵상 시 후원/기부 뱃지 부여 (Retention & Monetization)
3. **SNS 특화 묵상 이미지 생성**: 오늘의 구절과 AI 묵상 요약을 인스타 스토리 최적화 비율로 추출 (Viral)
4. **테마별 큐레이션 푸시 알림**: '위로가 필요할 때', '감사할 때' 등 상황에 맞는 구절 AI 추천 푸시
5. **개인 묵상 연말/월말 연말정산 리포트**: 통계 데이터를 바탕으로 유저의 성취 유도

**🎨 Product Designer (사용자 경험, 감성적 가치, 사용성에 초점)**
1. **절기 달력 연동 다이내믹 테마**: 사순절, 부활절 등 절기에 따라 UI 테마 및 앰비언트 자동 전환
