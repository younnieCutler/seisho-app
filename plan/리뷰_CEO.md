## 문제 정의

대한예수교장로회 20~30대 청년 신자들이 평일에 꾸준한 성경 묵상 루틴을 유지하기 어렵다.
"주일에만 독실한 기독교인"이 되는 경향이 있고, 절기의 의미를 일상에서 연결하는 도구가 없다.

기존 앱(매일성경, 생명의삶)은 개인 읽기 중심이며, 교회 절기 맥락을 구조화하거나
소그룹과 함께하는 공동 루틴을 지원하지 않는다.

---

## What Makes This Cool

1. **절기 맥락 자동화** — 앱이 오늘이 사순절 며칠째인지, 대림절인지 알고 콘텐츠를 바꾼다.
2. **소그룹 공동 루틴** — 교회 소그룹 코드 하나로 같은 말씀을 보고, 1문장씩 나눈다.
3. **잔디 그래프** — 개인 기록이 시각적으로 쌓인다. 신앙의 흔적이 눈에 보인다.

## Cold Start 문제 해결 전략

- **지인 중심 런칭**: 본인 교회 소그룹 1~2개에 직접 배포
- **1인 사용 모드**: 소그룹 없이도 개인 루틴으로 완전 사용 가능
- 소그룹 기능은 발견 요소, 필수 요소가 아님

## 기술 스택 결정

| 항목 | 결정 | 이유 |
|------|------|------|

## 오프라인 전략

- 말씀 본문: SQLite 번들 (앱 내 영구 포함)
- 로컬 유저 데이터: MMKV (스트릭, 진도, 체크)
- 완료 체크 + 잔디: 기기 로컬 → 온라인 복귀 시 Supabase 동기화
- 동기화 충돌: last-write-wins (UTC timestamp)

---

## 소그룹 코드 규격

- 6자리 영숫자 랜덤 (예: `AB12XY`)
- 전역 유니크, 재사용 불가 (soft-delete)
- 잘못된 코드 5회 입력 시 10분 잠금

## 성공 기준 (v1 배포 후 4주)

- WAU 10명 이상 (한 주에 ≥1회 체크인)
- 루틴 완료율 40% 이상 (한 주에 3일 이상 체크)
- 소그룹 나눔 참여율 30% 이상

## 빌드 순서 (권장)

1. 성경 데이터 준비 (Open Bible ko-KR → ko.db, 절기 JSON)
2. Expo 프로젝트 초기화 + Supabase 셋업
3. 홈 화면 (오늘의 절기 + 말씀)
4. 완료 체크 + 잔디 그래프
5. 소그룹 코드 + 피드
6. 온보딩 플로우 (마지막)
